5. 근본 원인 요약

Weight 계산 파이프라인의 병목

  • BASE_WEIGHT(0.12)가 대부분의 TYPE_CAP(A:0.15, B:0.12, C:0.10, D:0.20)보다 크거나 같아서, SCORE_FACTOR 1.0 이상인 액션(S_BUY, TREND_BUY, HOLD)은 모두 CAP에 잘림
  • 결과적으로 6단계 액션 체계(S_BUY / TREND_BUY / HOLD / REDUCE / EXIT / STOP_EXIT)가 실질 2단계(포함=0.10 / 감축=0.084)로 퇴화
  • BUY threshold 변경은 S_BUY↔HOLD 전환만 유발하는데, 이 두 액션의 최종 weight가 동일(0.10)

참고: REDUCE → EXIT 전환은 영향 있음

BUY=66일 때 REDUCE가 10,511건으로 증가하고 Trade도 4,238건으로 4배 증가. 이는 HOLD→REDUCE(weight 0.10→0.084)와 REDUCE→EXIT(weight 0.084→0) 전환이 유일하게 weight에 영향을 주기 때문. 다만 이 차이도 manage_portfolio() 정규화에서 대부분 희석됨.
